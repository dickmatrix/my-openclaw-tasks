# Intent Engine - 意图识别与任务分解

## 核心能力

### 1. 意图识别 (Intent Recognition)

当用户输入模糊诉求时，自动识别：

| 输入特征 | 识别类型 | 处理策略 |
|----------|----------|----------|
| "做X" / "帮我X" / "要X" | 执行类 | 分解为具体步骤，直接执行 |
| "怎么X" / "如何X" / "什么X" | 询问类 | 先搜索，再回答，必要时分解执行 |
| "帮我看看X" / "检查X" | 检查类 | 执行检查任务，报告结果 |
| "能不能X" / "可以X吗" | 确认类 | 分析可行性，给出建议 |
| 多个不相关需求 | 批量类 | 分解为独立任务，并行执行 |

### 2. 任务分解 (Task Decomposition)

**分解原则**:
- 每个子任务必须有明确输出
- 子任务之间保持最小依赖
- 先完成能独立执行的部分

**分解模板**:
```
原始需求: [用户的模糊输入]
├── 子任务1: [具体可执行的任务] → [预期产出]
├── 子任务2: [具体可执行的任务] → [预期产出]
└── 子任务3: [具体可执行的任务] → [预期产出]
```

### 3. 执行模式

| 模式 | 触发条件 | 执行方式 |
|------|----------|----------|
| 同步执行 | 用户要求立即完成 | 直接执行，实时反馈 |
| 后台执行 | 用户说"去弄" / "后台处理" | spawn subagent执行 |
| 定时执行 | 用户说"晚点" / "等会" | 设置cron延迟执行 |

### 4. 模糊输入处理流程

```
用户模糊输入
    ↓
意图分类 (Intent Classification)
    ↓
需求澄清 (如果太模糊)
    ↓
任务分解 (Task Decomposition)
    ↓
生成执行计划 (Execution Plan)
    ↓
自动执行 / 确认后执行
```

## 意图识别规则

### 关键词模式匹配

```
执行意图: 做|帮我|给我|要|搞|弄|设置|配置|创建|开发|写|编译|运行
查询意图: 怎么|如何|什么|哪里|谁|为什么|多少|何时
检查意图: 看看|检查|验证|测试|看一下|查一下|诊断
确认意图: 能不能|可以|是否|行不行
```

### 隐含意图推断

当用户说"X坏了" → 自动识别为"检查X问题"当用户说"X很慢" → 自动识别为"检查X性能"
当用户说"帮我弄个Y" → 分解为"理解Y需求 → 创建Y → 验证Y"

## 任务状态

| 状态 | 含义 |
|------|------|
| pending | 待执行 |
| running | 执行中 |
| completed | 已完成 |
| blocked | 阻塞(等待前置) |
| failed | 失败 |

## 使用方式

在响应用户前，先调用此引擎：

```markdown
1. 读取用户输入
2. 调用 intent_engine() 识别意图
3. 如果太模糊 → 生成澄清问题
4. 如果可执行 → 分解为子任务
5. 执行并跟踪状态
```
