# Autonomous Evolution System - 自主进化系统

## 目标

像Cursor一样工作：
1. **自动识别**模糊诉求的真实意图
2. **丝滑分解**为可执行的子任务
3. **无缝执行**调用系统资源
4. **持续进化**从每次交互中学习

---

## 系统架构

```
用户输入
    ↓
┌─────────────────┐
│ Intent Engine   │ ← 意图识别 + 任务分解
└─────────────────┘
    ↓
┌─────────────────┐
│ Task Planner    │ ← 生成执行计划
└─────────────────┘
    ↓
┌─────────────────┐
│ Executor        │ ← 自动执行
└─────────────────┘
    ↓
┌─────────────────┐
│ Evaluator       │ ← 结果评估 + 学习
└─────────────────┘
    ↓
进化循环
```

## 核心模块

### 1. Intent Engine (意图引擎)

**职责**:
- 识别用户真实意图
- 处理模糊/不完整输入
- 生成澄清问题(如需要)

**规则库**:
- 动词 → 动作映射
- 名词 → 实体识别
- 上下文推断

### 2. Task Planner (任务规划器)

**职责**:
- 将意图分解为具体任务
- 确定任务依赖关系
- 排列执行优先级

**输出**:
```markdown
## 执行计划

### Task 1: [任务描述]
- 类型: [execute/research/check]
- 工具: [需要的工具]
- 输出: [预期结果]
- 依赖: []

### Task 2: ...
```

### 3. Executor (执行器)

**职责**:
- 按计划执行任务
- 调用合适工具
- 处理执行异常

**执行模式**:
- 同步: main session直接执行
- 异步: spawn subagent执行
- 批量: 并行执行独立任务

### 4. Evaluator (评估器)

**职责**:
- 验证任务完成度
- 收集执行反馈
- 识别改进点

**学习机制**:
- 成功模式 → 记录到MEMORY.md
- 失败模式 → 记录到error_log
- 重复需求 → 优化执行路径

---

## 进化循环

### 每次执行后自检

```
1. 任务完成度: 100%? 如果不是，为什么？
2. 执行效率: 能更快吗？步骤能更少吗？
3. 用户反馈: 是否需要澄清？意图理解正确吗？
4. 模式识别: 这是重复需求吗？能自动化吗？
```

### 每日进化任务

| 时间 | 任务 | 输出 |
|------|------|------|
| 启动时 | 检查SESSION-STATE.md | 了解当前状态 |
| 执行后 | 更新MEMORY.md | 记录学习 |
| 每天 | 回顾error_log | 修复问题 |
| 每周 | 优化执行模板 | 提升效率 |

## 意图理解示例

### Example 1: 模糊输入

**用户**: "帮我弄个网站"

**Intent识别**:
```
输入: "帮我弄个网站"
类型: 执行类
置信度: 高
隐含实体: [网站类型未知, 目的未知]
```

**任务分解**:
```
├── Task 1: 了解网站用途 → 澄清问题
├── Task 2: 确定技术栈 → 技术选型
├── Task 3: 创建项目结构 → 代码文件
└── Task 4: 验证可运行 → 测试
```

**澄清问题**: "什么样的网站？个人主页/博客/电商/作品集？"

---

### Example 2: 半模糊输入

**用户**: "我的网站打开很慢"

**Intent识别**:
```
输入: "我的网站打开很慢"
类型: 检查类
实体: [网站: 未指定]
隐含意图: 性能诊断
```

**任务分解**:
```
├── Task 1: 定位网站 → 获取URL/本地路径
├── Task 2: 性能检测 → lighthouse/core-web-vitals
├── Task 3: 分析瓶颈 → 网络/渲染/资源
└── Task 4: 生成报告 → 优化建议
```

---

### Example 3: 清晰输入

**用户**: "创建 `hello.py`，输出Hello World"

**Intent识别**:
```
输入: "创建 `hello.py`，输出Hello World"
类型: 执行类
实体: [文件: hello.py, 内容: Hello World]
置信度: 100%
```

**任务分解**:
```
└── Task 1: 创建hello.py → file created
```

## 使用指南

### 自动执行流程

1. 用户输入 → 自动触发Intent Engine
2. 如果需要澄清 → 生成问题
3. 如果可执行 → 分解任务并执行
4. 执行完成 → 更新状态 + 记录学习

### 手动触发进化

```bash
# 检查进化状态
cat SESSION-STATE.md

# 查看学习日志
cat memory/YYYY-MM-DD.md
```
